Vibey — permission model

Three tiers. Postgres RLS enforces it. Vibey only sees what the asker can see.

Flow — how a request gets filtered

sequenceDiagram autonumber participant Asker as Asker
(site visitor / member / admin) participant Edge as chat-with-vibey participant DB as Supabase + RLS participant LLM as Claude via OpenRouter Asker->>Edge: message + JWT (or none) Edge->>DB: query context WITH asker's JWT Note over DB: RLS policies filter every row
by tier: public / member / admin DB-->>Edge: only rows the asker may see Edge->>LLM: system prompt + filtered context + message LLM-->>Edge: reply (can only reference what was loaded) Edge-->>Asker: streamed reply Edge->>DB: log to agent_chat_logs (service role, admin-only)
